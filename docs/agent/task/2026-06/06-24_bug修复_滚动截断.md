## 1. 需求

上一轮修了"整页滚"问题,但矫枉过正:把 `html, body, #app` 锁了 `height: 100vh + overflow: hidden` 之后,如果内部链条某处高度算不对,内容就被截断无法滚动。本轮回退那条改动,改成 SkillsView 内部自取一屏高度,body 在极端情况下兜底可滚。

## 4. 问题与方案

**问题:上一轮矫枉过正导致内容被截断。**

锁 `html, body { height: 100vh; overflow: hidden }` 是"双刃剑"——只有当内部链条高度都算对才不截断。`SkillsView .skills-layout` 用 `flex: 1` + grid,在没有确定高度的父级里(grid item min-height:auto),会被子内容撑大,撑大后 overflow:hidden 截断。

方案:**回退锁 body,改成 SkillsView 自身 `height: calc(100vh - 88px)`**。topbar ≈ 46px(高 py-3 = 24px + 内容 ≈ 22px),content-area p-5 = 40px,合计 88(保守值)。
- 99% 情况下 body 不滚
- 内部 grid 用 `grid-template-rows: minmax(0, 1fr)`,子项显式 `min-height: 0` 避免撑大
- 极端小屏(< 88 + 一屏内容)时 body 兜底滚(不锁)
