## 4. 问题与方案

### 4.1 `pkg/task/run_test.go` 600s 超时

`go test ./...` 跑全包时,`ginp-api/pkg/task` 包里有个老的 cron 测试 600s 超时,导致整个 `go test` exit 非 0。
- **方案**:只跑本次改动相关的包(`./internal/gapi/controller/skillbox/cproject/...` / `./internal/gapi/service/project/...` / `./internal/skilladapter/...`),三个全 ok。pkg/task 的超时是历史遗留,与本次改动无交集,不动。

### 4.2 `api-server/cmd/web/frontend/` 不是 vite 部署目录

`docs/agent/memory/project.md:16` 写"`api-server/cmd/web/frontend/dist/` 是 web 单进程部署用的",但实际该目录里只有 wails 桌面 build 产物(`Inter-Medium.ttf` / `wails.png` / `style.css` 等),不存在 `vite.config.js`,`npx vite build` 会报 `CustomEvent is not defined`。
- **方案**:本次只同步 `frontend/dist/`(即 vite 标准产物);web 单进程的同步机制需要后续单独确认,不混在本次。

### 4.3 `platform.fs.reveal` 桌面 / Web 差异

桌面端 `reveal` 实际是 wails hook(`/api/desktop/fs/reveal`);Web 端后端返 501 + `fallback_url`,platform 内部用 `window.open` 兜底。
- **方案**:`openInFinder` 直接 `await platform.fs.reveal(p.root_path)`,不分支;失败时 toast 错误。不在 controller 层额外判断 runMode,platform 抽象已经处理好(Web 端至少能 fallback 到 file:// 链接)。
