# 桌面托盘图标化 + 左键弹菜单

**日期:** 2026-06-29
**状态:** 已完成
**Commit:** `62040ec feat(desktop): 托盘图标化并改为左键弹菜单`

## 1. 需求

靓仔要求优化桌面端系统托盘,两项改动:

1. **去掉"Skill Box"文字,改用图标** — 当前 `t.SetLabel("Skill Box")` 显示的是文字,期望显示一个图形图标
2. **左键点击托盘图标 = 弹出托盘菜单** — 当前左键和双键都触发显示主窗口,期望左键弹菜单(更符合 macOS 用户对菜单栏的肌肉记忆)

## 2. 关键决策

### 图标资源策略

- **方案选定**:运行时动态生成图标,从 `//go:embed` 的 `build/appicon.png` 缩放而来,不维护额外美术资产
  - **macOS 模板图标**:36×36,黑色 + 源图 alpha(系统会随 Light/Dark 模式自动反色)
  - **Win/Linux 彩色图标**:32×32,直接 RGBA 缩放
- **算法**:`golang.org/x/image/draw.CatmullRom.Scale`(高质量缩放)

### 跨平台行为矩阵

| 平台 | 图标 API | Label | OnClick | OnRightClick |
|---|---|---|---|---|
| darwin | `SetTemplateIcon` | `""`(空) | `OpenMenu` | 不设(默认右键弹菜单) |
| windows / linux | `SetIcon`(彩色) | `"Skill Box"` | `OnShow` | `OpenMenu` |

**SetTemplateIcon 和 SetIcon 在 macOS 上互斥**(darwin 的 `setIcon` 读 `isTemplateIcon` 标记,两个都调会污染),必须按平台二选一。

### 砍掉 OnDoubleClick

Wails v3 alpha.60 的 darwin 路径 `processClick`(`systemtray_darwin.go:255-287`)根本不 dispatch `doubleClickHandler`(只 case left/right button)。**macOS 上双击永远不触发**,留着 `OnDoubleClick` 是误导性死代码。

主窗口的入口改为:
- 菜单项"显示主窗口"(已存在)
- 全局快捷键 `Cmd+Shift+S`(已存在)

### embed 路径处理

- `//go:embed` **不支持符号链接**(`pattern appicon.png: cannot embed irregular file`)
- **采用**:把 `build/appicon.png` 复制成 `desktop/appicon.png` 实体文件,`//go:embed appicon.png` 把图标编进二进制,运行时不再依赖磁盘路径

## 3. 实施过程

### 第一版:alpha 阈值二值化(失败,已修正)

最初用 "alpha > 128 的设为黑色" 硬阈值,结果图标主体几乎透明。**根因**:源图 `build/appicon.png` 实际是"透明背景 + 深灰 W 字母"(34.7% 全透明 + 59.9% 不透明但 RGB≈30/30/30),不是"黑底白字"。MiniMax 描述"黑底白字"是 viewer 渲染透明区为黑色的视觉假象。

### 修正版:RGB 压黑 + 源图 alpha 直接沿用

```go
for y := 0; y < size; y++ {
    for x := 0; x < size; x++ {
        _, _, _, a := tmplImg.At(x, y).RGBA()
        // CatmullRom 缩放后已含抗锯齿过渡(alpha 中间值)
        tmplImg.SetNRGBA(x, y, imgcolor.NRGBA{R: 0, G: 0, B: 0, A: uint8(a >> 8)})
    }
}
```

**验证结果**(脚本生成 36×36 PNG 后统计):
- 33.5% 完全透明(alpha=0,背景)
- 55.9% 完全不透明黑色(W 主体,RGB=0/0/0)
- 10.6% 半透明(抗锯齿过渡)

## 4. 踩坑

- **`//go:embed` 不支持符号链接** — 软链接方案失败,改用 `cp` 复制实体文件
- **golang.org/x/image 不是 wails 的传递依赖** — Plan agent 推断错了,需要 `go get golang.org/x/image/draw` 显式加
- **`image/color` 与 `application/color` 命名冲突风险** — 用 `imgcolor` 别名
- **alpha 阈值不能直接用源图亮度** — 源图主体 RGB 接近 30(深灰),转灰度后 alpha≈30 几乎透明;模板图标应保留源图 alpha 形状,RGB 压黑
- **Wails darwin 路径的 OnDoubleClick 是死代码** — 源码确认,留着误导
