## 1. 需求

用户原话:
> '/Volumes/MyDrive/Home/dicoder/projects/skill-box/frontend/src/views' 前端界面中个所有需要操作的地方都改成弹窗,不要使用跳转新页面,比如新建技能 / 对技能打标签/编辑技能等

- 共用基础设施:通用 Modal 组件、通用确认弹窗(取代原生 `confirm()`)

## 4. 问题与方案

- **原生 confirm() 阻塞问题**:浏览器 `confirm()` 在 Wails 桌面端可能行为异常(阻塞主进程),而且无法定制样式。改用通用 Modal 包一个确认弹窗,通过 Promise 异步确认。三个 view 各自封装,不复用抽象(3 个文件,3 个 state,改动可控)。
- **AI 侧栏的 `currentSkillMd` 计算属性**:依赖 `editorOpen`,改成弹窗后仍然响应式,无需改逻辑。
- **Tag 弹窗的关闭清理**:`tagMessage / tagError / diffResult / newTagName` 等状态,关闭时清空,避免下次打开还显示上次的提示。
- **edit 时的 `error` 共用问题**:SkillsView 的 `error` ref 同时被表格上方的错误条和 editor 弹窗共用;现在 editor 弹窗内另起一块展示,关闭弹窗时不清空,主视图的错误条继续工作。
