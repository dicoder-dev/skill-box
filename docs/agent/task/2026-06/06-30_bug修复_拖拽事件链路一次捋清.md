## 4. 问题与方案

### 4.1 真正的根因 — 把"DOM 自然路由"搞复杂了

之前 4 次 commit(`589aada` 等)累积的修复方向都是"让边缘 drop 穿透到根",但实现方式是**人造一个 40px 边距检测 + 早 return**。这完全违背了 DOM 事件冒泡的本来语义:

- 事件落在 `.tree-row` 内 → TreeNode 的 @drop 处理
- 事件落在 `.tree-container 空白处`(没命中任何 .tree-row)→ 冒泡到 `.tree-container.onTreeContainerDrop`

DOM 天然就做了这件事,我之前非要造一个"边缘检测",反而引入了一堆 bug:
- 鼠标贴近容器边缘 + 想拖到某个具体 group → 被错误穿透到根分支
- target=null 走"合成 target"路径,但合成 target 的解析有时和真实 target 行为不一致

**修复方案**:TreeNode.onDropGroup 恢复 e.stopPropagation(本节点命中就吃掉,冒泡给 .tree-container 的就是"没命中任何节点的"事件),删掉 ROOT_DROP_MARGIN 和 isRootDropIntent。SkillsView.onTreeContainerDrop 去掉 stopPropagation(纯粹清理)。

### 4.2 onTreeDrop 加防御

加 `if (!source.path) return` 防御空 source.path(异常数据传输出错,不该弹错给用户)。整体拦截顺序加了注释,说明每条规则的意图,避免后人再写一个补丁把已拦截的 case 又放开。
