## 4. 问题与方案

**问题 1:滚动穿透。**
根因:
- html/body 是 `min-height: 100vh`,内容超过 100vh 时整页 body 撑高,浏览器滚 body
- `.main-content` 是 flex col,没 `min-h-0` → flex 子项最小高度默认 min-content,可能撑高
- `.skills-layout` 用 `height: 100%` 在 flex 父中脆弱(拿到的是 0 或内容高度)

方案:
- html/body 锁 100vh + overflow:hidden,根除 body 滚动
- `.main-content` 加 `min-h-0`(tailwind class)
- `.content-area` 显式 `h-full` + `min-height:0`
- `.skills-layout` 改 `flex: 1` + `grid-template-rows: minmax(0, 1fr)`,不再依赖 `height: 100%`

**问题 2:编辑弹窗冗余。**
原 startEdit() 打开 Modal 让用户改 frontmatter + body。新需求"在当前界面把渲染内容改成可编辑的编辑器" — body 是主要内容,且已经渲染,只需把 body 区切换到 textarea。frontmatter 暂不暴露(改 frontmatter 不算高频需求,需要时再扩展)。

方案:
- `editing` 状态控制 detail-body 切换
- `editBody` 双向绑 textarea
- 保存时 `rebuildSkillMd(newBody)` 用现有 frontmatter 拼回 SKILL.md,调 updateSkill
- 切换 / 删除 skill 时 `cancelInlineEdit()` 清掉,避免残留
