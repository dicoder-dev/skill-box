**日期:** 2026-06-25
**状态:** 已完成

## 1. 需求

`docs/agent/task/2026-06/README.md` 当前文件结构里没有显式的"改动的文件"栏目,
AI 写 task 时通常只在 `## 7. 总结` 的"留下了什么"小节零散地提到几个文件路径,存在以下问题:

- 不强制 → 时常漏写,事后无法靠 task 文档 grep 到所有改动文件
- 不分类型 → 新增 / 修改 / 删除混在一起,不清楚是新增还是改动
- 不在任务过程中维护 → 一般是任务结束才补,中途任何一轮出问题都查不到当前已动过哪些文件

用户希望:在 task 文件结构里新增一个 **"8. 改动的文件"** 栏目,要求 AI 在每一轮任务过程中就同步维护,
并按"新增 / 修改 / 删除"分类列出完整路径 + 一句话说明,任务结束填总结前要复核。

## 4. 问题与方案

**问题:`## 7. 总结` 里的"留下了什么"已经能列文件,为什么还要新增 `## 8. 改动的文件`?**

方案:不替代 7 节,而是分工:
- `## 7. 总结` 写"这次任务完成的事 / 留下的决策 / 给下次的提醒",文件清单是副产品
- `## 8. 改动的文件` 写"纯清单",强制分类型、强制完整路径、强制在任务过程中就维护,
  便于中途接手的人/未来的 Claude 通过 `grep` 精准锁定"这次改动了什么"
两份内容有重叠无害:7 节是叙述,8 节是事实清单。

**问题:不涉及文件改动的任务(纯讨论/纯澄清)是否要写 8 节?**

方案:仍要保留小节占位,内容写"本任务为纯讨论 / 纯澄清,未涉及任何文件改动"。
这样所有 task 文件结构对齐,不会出现"有的有 8 节、有的没有"的混乱。
