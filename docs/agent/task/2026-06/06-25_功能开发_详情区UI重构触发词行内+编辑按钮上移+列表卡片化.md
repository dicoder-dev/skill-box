## 4. 问题与方案

### 编辑按钮位置
- 旧位置:detail-body 顶部 section-header 右侧
- 新位置:detail-title-row 右侧(LOCAL 徽章之后)
- 实现:用 `.detail-title-actions` 容器,`margin-left: auto` 推到行尾
- 查看态显示"编辑"按钮,编辑态不显示(因为"取消/保存"在 body 顶部)

### description 移到 title-block 内
- 旧:detail-toolbar 内只有标题 + 描述,p 元素在 title-row 之后
- 新:同样在 title-block 内,只是变成可切换的 p / textarea
- `v-if="!editing && currentMeta.description"` + `v-else-if="editing"` 互斥切换
- 编辑态 textarea 用 `.desc-editor` 样式(rows=2,可垂直 resize)

### 触发词行内展示
- 旧:独立 section 展示,meta-block + meta-label
- 新:detail-title-block 内的 `.detail-triggers-row`,label + 文本 / 编辑态 textarea
- 触发词为空时不渲染(查看态),编辑态永远渲染(因为可能要改)

### 编辑范围确定
- 用户选"触发词+描述+正文"(推荐)
- name / version / scope 走原来的"编辑弹窗"
- 这样改动小、风险低,符合日常改内容需求

### rebuildSkillMd 签名
- 旧:`rebuildSkillMd(newBody, newTriggers)`
- 新:`rebuildSkillMd(newBody, newTriggers, newDescription)`
- newTriggers / newDescription 都是可选,默认仍读 currentMeta,向后兼容
