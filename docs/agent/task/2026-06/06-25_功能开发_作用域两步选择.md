## 4. 问题与方案

**Q1: toast 出现在 chip 状态还没刷新时,会视觉错位**
A: 把 `toast.success + flashTarget` 放在 `await loadScopeStatus()` 之后。
loadScopeStatus 完成后 scopeHits 才更新,selectedToolHitExists(tg) 才能正确反映
新状态。同步做的话,flash 那 2s 内 chip 还是旧的 muted 态 → 看起来"高亮了一个未启用的 chip"。

**Q2: handleScopeChipClick 里已经 `await loadScopeStatus` 过一次,doApplyOne 内部又 await — 会重复**
A: 把 loadScopeStatus 的职责收归 doApplyOne / doUnapplyOne,handler 不再重复调用。
职责分层:handler 负责"判断走哪条分支 + 弹确认",do* 负责"调接口 + 刷新 + 反馈"。

**Q3: chip-flash 动画用 transform:scale 会不会跟 chip 内的 icon 列表布局冲突?**
A: 不会。动画作用在父 button 元素上,内部子元素跟随父级 transform 一起缩放,
整体一致;脉冲 1.6s 一次,scale 峰值 1.04,视觉上轻微"跳一下"而不是真的变大,避免布局抖动。
