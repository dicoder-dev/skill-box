# 作用域区域改为"工具单选 + 作用域联动"

**日期:** 2026-06-25
**状态:** 进行中

## 1. 需求

首页 SkillsView 的 scope 区域当前是**两个独立维度**:工具行批量、作用域行批量。
用户希望改成**级联两步选择**:
- 第一步:点工具行 chip,工具进入"选中"态(蓝色边框 + 高亮)
- 第二步:点作用域行 chip(全局 / 各项目),只对**当前选中工具**的 (scope, project) 单条组合做启用/停用
- 未选工具时,作用域 chip 置灰不可点,提示"先选工具"
- 工具行不再触发批量启用/停用(改由作用域 chip 完成)

## 4. 问题与方案

**Q1: 选中工具的"已生效"态如何跟"已选中"态共存?**
A: 用 `box-shadow: 0 0 0 1px var(--accent-blue)` 模拟双层边框,避免 border-width 变化导致尺寸跳动。
- 命中 + 选中 → 主色填充 + 蓝色外框
- 未命中 + 选中 → 虚线变实线 + 蓝色外框

**Q2: 后端 hits 是否覆盖了所有 (tool, scope, project) 笛卡尔积?**
A: 是。后端对每个 adapter 都生成 global + project 候选命中,前端 scopeTargets 聚合后,**任何位置都会有 chip 出现**,不会出现"某工具从未生效过 → chip 缺失"的情况。
fakeHit 兜底分支保留作为防御,正常情况不会触发。

**Q3: 工具行 chip 的 busy 状态现在还需要吗?**
A: 不需要。工具行点击只切选中态,不再触发耗时操作,busy 状态移除以减少视觉噪音。
