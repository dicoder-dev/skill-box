## 4. 问题与方案

**现象:** 编辑态两个 textarea 视觉高度差不多,触发词被撑到 ~2.8 行高,看起来跟描述框一样
**定位:** `.triggers-editor` min-height 56px 是早期为了"两个框视觉对齐"留的旧值,跟 `rows="1"` 不匹配
**方案:** 改 min-height 到精确的 1 行 + padding = 36px
**教训:** `rows` 是 HTML 自带的初始行数属性,min-height 应该跟 rows 对齐精确计算,而不是拍脑袋设个 56px 凑数
