## 4. 问题与方案

### 问题 1: 列表"已应用工具"如何拿?
- 方案选择: 前端 N+1 调 scope-status(每个 skill 一次,慢) / 后端 list 接口注入字段(快)。
- 选 后端注入。理由:文件 stat 很轻,200 个 skill × 5 个 adapter × 1 stat ≈ 1000 次 stat,
  在用户可感知阈值内(应该 < 200ms);避免 N+1 拥堵。
- 实现: 复用 scope_status.a.go 的扫描逻辑,抽 `GlobalAppliedTools(name)` 工具函数,
  list_skills.a.go 给每个 item 加 `applied_tools: []string` 字段。

### 问题 2: "未选工具"时 chip 不该置灰,但要明确提示
- 旧行为: `<button :disabled="isScopeTargetDisabled(tg)">` + `.chip-scope-target:disabled { opacity: 0.45 }`。
  disabled 触发 native grayout,在 macOS / 各种浏览器都很重,看起来"完全不可点"。
- 新行为: 删 disabled,加 `chip-target-no-tool` class(opacity 0.85),保留 hover 反馈;
  点击 → 弹 toast.info(scopeSelectToolFirst),告诉用户"先选工具"。
  这样视觉上"正常可点",点完有明确反馈。

### 问题 3: "未启用工具" muted 不要降透明度
- 旧: `.chip-tool.chip-muted { opacity: 0.7 }` 整体变灰,看起来像 disabled。
- 新: 去掉 opacity,只保留虚线 border + 浅色文字区分"未启用 vs 已启用",
  实际可点。
