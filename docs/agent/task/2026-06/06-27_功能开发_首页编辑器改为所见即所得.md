## 4. 问题与方案

### 4.1 编辑器与现有数据模型的对齐

- 问题:opcbase 用 Tiptap 存 HTML,而 skill-box body 字段是 markdown 字符串
- 方案:RichTextEditor 内部做双向转换 — 输入用 renderMarkdown(md)→HTML 喂 Tiptap,onUpdate 时用 htmlToMarkdown(html)→md 回写
- 收益:不破坏现有 rebuildSkillMd / buildSkillMd 的接口,改两处 v-model 即可

### 4.2 htmlToMarkdown 单元自测

- 问题:浏览器 DOMParser 在 Node 跑不了
- 方案:用 linkedom(纯 JS DOM 实现,~100KB)做桩,跑 13 个用例验证序列化正确
- 教训:不引 turndown(90KB+) — 我们的需求简单,自写 ~150 行就够

### 4.3 blockquote 多 P 子节点的换行

- 问题:最初 BLOCKQUOTE 里的两个 `<p>` 之间产生 `\n\n`,加 `> ` 前缀后多一行空 `>`
- 方案:blockquote 内部把多空行压成 `\n` 再 split,实现紧凑的"每行一个 P"映射
- 教训:markdown 引用语法对子块间距容忍度高,但用户期望紧凑(1 行 = 1 P)

### 4.4 测试桩差异

- 问题:linkedom 不会把 `\\`` 解释为反引号(浏览器 DOMParser 会)
- 方案:测试用例改用 `&#96;` HTML entity,绕开 linkedom 解析差异
- 教训:写跨运行时测试时,涉及转义字符最好用 HTML entity 表达,而不是直接写转义序列
