## 4. 问题与方案

### 问题
- 之前列表项的 applied_tools 来源是 `listSkills` 接口(后端注入);
  右侧 doApplyOne / doUnapplyOne 操作成功后只调 loadScopeStatus 刷了
  scopeHits(scope-status 实时扫描磁盘),**没刷 items**。
- 结果:用户右侧"启用 Claude"成功后,scopeHits 里 Claude exists=true,
  但 items[].applied_tools 仍是旧值,左侧列表项不显示新加的工具 chip。

### 方案
- 不重拉整个列表(避免 loading 闪烁 + 200 skill × 5 adapter 的 stat 开销),
  改用**本地局部更新**:
  - `localGlobalAppliedTools()`:从已有 scopeHits 推算 global exists 的 tool_id 集合
  - `syncLocalAppliedTools()`:在 items 里按 skillKey 找当前 skill,
    浅拷 + 替换 applied_tools 字段
  - 在 doApplyOne / doUnapplyOne 成功路径(loadScopeStatus 之后)调一次
  - 在 loadCurrent 切 skill 时调两次:清 scopeHits 前(同步旧 skill) +
    拿到新 scopeHits 后(同步新 skill,防御性)

### 教训
- 上次加 applied_tools 字段时只考虑了"首次加载"和"搜索刷新",
  漏了"右侧操作后局部更新"。下次加 server 端聚合字段,
  要在所有客户端 mutation 路径都加本地同步逻辑。
