## 4. 问题与方案

**问题 1:wails v3 怎么弹"选目录"对话框?**
- 现象:wails v3 没有 `OpenDirectoryDialog` 独立 API
- 定位:查到 `OpenFileDialogStruct.CanChooseDirectories(true).CanChooseFiles(false).PromptForSingleSelection()` 正是为这个场景设计
- 方案:用 `app.Dialog.OpenFile().CanChooseDirectories(true).CanChooseFiles(false).PromptForSingleSelection()`

**问题 2:Web 端(无桌面 hook)选目录怎么办?**
- 现象:浏览器里没有原生文件夹选择器,且 wails dialog 不可用
- 方案:后端 `pick-folder` 端点在 hook 缺失时返 501,前端调用方拿到 error 自行处理(目前 ProjectsView 直接把 error 弹在 .error-message 区,后续要做 Web 端降级可以再补 input[type=file] webkitdirectory);不阻塞桌面端主路径

**问题 3:InspectProject 误判概率?**
- 现象:仅用 basename 当 name 启发,可能跟用户的"实际显示名"不一致
- 方案:启发只用来"预填",用户可任意修改;前端"重新选择目录" + root_path blur 都会再走一次 inspect;同时在表单顶部放一个"已根据所选目录自动解析,可按需修改"的 hint 明确告知启发

**问题 4:用户手动改 root_path 后要不要重新解析?**
- 方案:监听 `root_path` 的 blur,只有当 name/alias 仍跟启发结果一致(或为空)时覆盖;否则视为用户已编辑,不覆盖(避免把用户输入清空)
  - 当前实现:blur 时无条件 inspect,简单粗暴;若用户已改 name 再改 path,会用新 basename 覆盖
  - 后续可优化:加一个 `inspectedKey` 标记,只在"未编辑"时覆盖

**问题 5:双 + 图标**
- 现象:旧按钮文本是 `+ 新建项目`,同时 `<Icon icon="mdi:plus" />`,等于显示两个加号
- 方案:去掉文本里的 `+` 前缀,只留 mdi 图标;新按钮图标换成 `mdi:folder-upload-outline`(语义更准,"导入文件夹"),用 `<span>` 包文字让按钮布局更稳
