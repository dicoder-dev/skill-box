# 桌面端初始窗口按屏幕宽度 80% 自动适配

**日期:** 2026-07-02
**状态:** 已完成

## 1. 需求

用户反馈:桌面端主窗口初始尺寸固定为 1280×800,在 4K/2K 大屏上显示偏小,
希望初始宽度按屏幕宽度的 80% 计算。

最终决定:
- 宽度 = 屏幕 DIP 宽度 × 80%
- 高度 = 宽度 × 5 / 8(保持 16:10 视觉比例)
- 不做尺寸记忆(本次不做,后续单独迭代)
- 由用户自己 `wails3 dev` 验证

## 4. 问题与方案

### 问题 1:GetScreen 调用时机

- 现象:`GetScreen()` 内部走 `InvokeSyncWithResultAndError(w.impl.getScreen)`,
  需要窗口已经 ready(Native 端已注册)。
- 方案:把"按屏幕比例调整"放到 `startupAsync()` 协程里(已经在等 500ms 让 Wails 主循环 ready),
  调 `w.SetSize(newW, newH)` 与 `w.SetMinSize(minW, minH)` 即可。
- 兜底:屏幕尺寸获取失败时,log warning,不动窗口,由 NewApp 的兜底 1280×800 顶住。

### 问题 2:不能覆盖用户的显式 cfg.Width

- 需求:如果调用方 (main.go) 显式传 cfg.Width,应当尊重它而不是按 80% 算。
- 方案:新增 `cfg.AutoSizeByScreen bool` 字段(默认 true),
  `NewApp` 里如果 `cfg.Width != 0 || cfg.Height != 0` 则把 `AutoSizeByScreen = false`,
  并把 `autoResize` 透传到 `App.startupAsync(autoResize)`。
- 当前 main.go 不传 Width/Height,直接走默认开启路径;后续要固定窗口尺寸直接给 Width/Height 即可。

### 问题 3:MinWidth / MinHeight 兜底

- 现象:当前 MinWidth=960 / MinHeight=600 是固定兜底。
- 决策:MinWidth = round(screenW × 0.6),且不低于 960;MinHeight = round(MinWidth × 0.625),
  且不低于 600。
- 实施:同样放到 startupAsync 里调 `SetMinSize`。
