## 4. 问题与方案

### 4.1 commit 切分
- 一开始按 plan 拆 4 个 commit(数据层 / 只读 UI / 写操作 UI / i18n 自测),但 ToolsView.vue 一开始就写完整(只读 + 写操作 + 删除确认),中间切 commit 反而容易冲突
- 实际切:**2 个功能 commit + 1 个 embed sync commit** = 3 commit 落地

### 4.2 v-for 中变量名与 i18n 函数冲突
- `const { t } = useI18n()` 占了 `t`;v-for 用 `t_item` 避免遮蔽
- 第一个 onToggleEnabled 写成 `t.enabled ? ... : ...`,把 `t` 当数据访问 — 修正为提前记录 `const willEnable = !t_item.enabled`

### 4.3 Vue 模板里 `button small class` 命名
- `<button class="ghost small">` 缺一空格无效;确保 class 数组写法

### 4.4 embed 同步源
- `api-server/cmd/web/frontend/dist/` 是 //go:embed 的 source-of-truth(被 git 跟踪)
- 用户用 git stash pop 后市场 install 相关文件被改名(rename),我最初一次 `git add dist/` 时如果用 `git add -A` 会一起 stage 用户的工作
- 修正:reset soft + 只点名 add dist/ 三个文件
