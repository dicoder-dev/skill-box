# 2026-07-01 Skill 市场拉取不全 + 改「安装」→「拉取」

## 1. 需求

靓仔提了两个问题:
1. **市场拉不到所有 skill**:`market_skills` 表里实际只有 26 条(skillhub 3 / skills.sh 23),全部是内置兜底 fallback,真实三方 catalog 没拉到。
2. **术语不准确**:市场里把"从三方源取到本地 store + 推到工具目录"叫"安装",但这个动作的本质是**拉取到 skill-box 统一管理**,"安装"在用户感知里更适合"从 apt/app store 装系统包"那种场景。

## 2. 决定

### A. 拉取不全

- **skills.sh 解析**:`Discover` 之前只跑 `parseOwnerRepoAtBody` 老版正则;改为同时跑 `parseOwnerRepoAtBody` + `parseHTMLLinks` 合并去重,两者都为空才走 `knownCatalogFallback`。
- **fallback 列表**:从 23 条扩到 30 条,新增 vercel-labs `next-best-practices`、obra `verification-before-completion`、dylnuge `vue-best-practices`/`react-best-practices`、anthropics `pdf`/`mcp-builder`/`frontend-design`/`skill-creator` 等常用 skill。`minCatalogFallbackSize` 从 20 提到 28。
- **skillhub 源**:站点暂未公开稳定 API,本来就只能走 `knownFallback`,短期无解,本次不动。

### B. install → pull 改名(范围已确认:前端 + 后端业务术语,保留 install alias)

#### 后端 service 层
- `Install` / `InstallV2` → `Pull` / `PullV2`(新名)
- `InstallInput` / `InstallResult` / `InstallV2Input` / `InstallV2Result` → `Pull*`(新名)
- `ErrInstallFailed` → `ErrPullFailed`(新名)
- 旧名全部留 alias(类型用 `type X = Y`,函数显式 wrap)

#### 后端 controller 层
- 文件 `install_skill.a.go` / `install_skill_v2.a.go` 重命名为 `pull_skill.a.go` / `pull_skill_v2.a.go`
- 函数名 `InstallMarketSkill` / `InstallMarketSkillV2` → `PullMarketSkill*`
- 类型 `Request*` / `Respond*` 同步改名,旧名留 alias
- HTTP 路径 `/api/skillbox/market/install` + `/install-v2` **保持不变**(向后兼容)
- `PermissionName` 改为 `skillbox.market.pull` / `skillbox.market.pull.v2`
- Swagger Title/Description 改为"pull"

#### 前端
- `core/store/market.js` action `install` → `pull`,`installing` → `pulling`,`lastInstallResult` → `lastPullResult`;`install` 留 alias
- `api/skillbox/market.js` `installMarketSkillV2` → `pullMarketSkillV2`,旧名留 alias
- `components/MarketInstallConfirm.vue` → `components/MarketPullConfirm.vue`(git mv);内部 class 名 `.install-*` → `.pull-*`;事件 `@confirm="(p) => market.pull(p)"`
- `views/MarketView.vue` import 同步,按钮文案改"拉取 / 再拉取一次",`market.installing` → `market.pulling`
- i18n:新增 `market.pullDialog` 段(中英),文案"拉取";"`market.installDialog` 留空占位,文件底部 `Object.assign` 把 `pullDialog` 同步进去,保证旧模板 `t('market.installDialog.*')` 仍能命中
- i18n 顶层:`errInstall` → `errPull`、`okInstalled` → `okPulled`、`btnInstall` → `btnPull`、`btnReinstall` → `btnRepull`;旧名留 alias
- en/zh 文案统一:"安装/Install" → "拉取/Pull"、"再装一次/Reinstall" → "再拉取一次/Re-pull"

#### 测试
- `market.s_test.go` 全部 `Install*` 改名 `Pull*`(测试名 + 类型引用 + service 调用)
- `cmarket_test.go` 不动(HTTP 路径没变)
- `pkg/task` 的 600s 超时、`pkg/cos` 的 STS 测试、`db/pgsql` 的 TestConnect 都是已有问题,与本次改动无关

- **HTTP 路由名 + 权限名变更**:`PermissionName` 改 `pull.v2`,旧用户数据里 `install.v2` perm 标记靠 service 层 `InstallV2` 薄包装 + `Object.assign` 兼容
- **别名污染**:`install` / `installDialog` / `InstallV2` 等 alias 散布,文件头注释已注明 "pull 是新名,install 留 alias",后续清理时统一删 alias
