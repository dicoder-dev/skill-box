# 首页导入弹窗新增"从本地 zip / 文件夹导入"

**日期:** 2026-07-01
**状态:** 进行中

## 1. 需求

用户原话：

> 首页导入覃川各种要新增支持从本地文件夹中导入 zip 压缩包或者文件夹，导入的时候要检查文件夹内是否存在 SKILL.md 文件

解构需求：

1. **入口位置** —— 首页（即 `SkillsView`）左栏的"导入技能"按钮（`onboarding.list.btnImportSkill`）目前弹 `OnboardingImportDialog`，里面只走"扫已装工具"流程。要在这个入口**新增**一种导入方式：「从本地 zip / 文件夹导入」。
2. **两种来源**
   - **本地文件夹**：用户选一个本地目录，里面应当存在 `SKILL.md`（支持单 skill 一级，也支持 Claude marketplaces 多级目录，下钻找到含 `SKILL.md` 的子目录）。
   - **本地 zip 压缩包**：用户选一个本地 `.zip`，解到内存 / 临时目录后，识别里面有 `SKILL.md` 的条目作为 skill（支持 `.skillbox` 旧包格式的兼容路径的解法）。
3. **校验** —— 导入前必须校验目录或 zip 里有 `SKILL.md`。**没有则报错**：前端弹 toast 提示"未找到 SKILL.md 文件"。
4. **写入路径** —— 复用现有 `skillimporter.Importer.Import` 逻辑（已经在 `PostOnboardingImport` 中验证可用，最终落到 `~/.agents/skills/<name>@<version>/`），不绕开 `sskill.Service.Create`，因此 `mskill` 表行会被正常 upsert，前端 `listSkills` 能正常拉到。

非目标：

- 不做 scan/import 二段式向导（即不复制 OnboardingView 的勾选 → 全部导入流程），新流程是「选文件 → 校验 → 直接导入 → 弹结果」。
- 不替换现有"扫工具"流程；只是并列新增一个 tab / 折叠区。

## 4. 问题与方案

> 开发中遇到的非平凡问题。

- **桌面端文件选择**：桌面端 `wails v3 OpenFileDialog` 用 `.Filter("*.zip")`，web 端 `<input type="file" accept=".zip">`。需要新增 `platform.fs.pickFile(opts)` 抽象（封装桌面 OpenFileDialog、Web FileDialog），同时为 `pickFolder` 也补 `.zip` 过滤器不必要（zip 由按钮文案区分）。
- **zip 来源**：桌面端直接拿绝对路径读盘；Web 端只能 FileReader 读 → multipart 或 base64。最简方案：Web 端 zip 也走 `<input type="file">` 上传到 `/api/skillbox/onboarding/import-local` 的 `application/octet-stream` 模式（复用 `pkg/import` 的字节流风格）。桌面端走 `{mode:"folder"|"zip", path:"/Users/.../x.zip"}` JSON。
- **多级目录**：Claude marketplaces 经常是 `~/.claude/plugins/marketplaces/xxx/skills/<name>/SKILL.md`。`walkSkills` 已有递归逻辑，但 `ReadSkillDir` 只读单层。新增的"按目录导入"要复用 walkSkills 找到所有候选。
- **`Importer.Import` 入参**：需要构造 `skillimporter.Report{FoundSkills:[]FoundSkill{Canonical, ToolID, SourcePath, ToolName}}`。ToolID 留空字符串可以让 importer 走"无工具"路径（即只写 store 不 apply），符合"导入本地"这个语义。
