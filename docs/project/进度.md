| `docs/project/{README,进度,开发报告}.md`                                              | 文档同步            |

**实现要点**

- 5 个 adapter 都基于 `BaseAdapter` 复用通用逻辑,各自只填 ID / DisplayName / Icon / DiscoverPaths 即可
- codex 路径覆盖三处:`~/.codex/skills`、`~/.codex/skills/.system`、`~/.codex/vendor_imports/skills/skills/.curated`;claude 走 plugin marketplaces 而非 `~/.claude/skills`;trae 走 `~/.trae/skills`;opencode / cursor 走默认 `~/.config/...` 和 `~/.cursor/...`(草案)
- `skillimporter` 抽象:Report = 一次扫描的完整产出(每个目录、每个 tool、每个 found skill);Import = 把 Report 里勾选出来的条目落到 skillstore 全局域
- Onboarding 三个端点:`GET /api/skillbox/onboarding/status` 报告 5 个 adapter 状态 + 上次 scan 摘要;`POST /api/skillbox/onboarding/scan` 跑扫描并把精简 Report 缓存到进程内单例;`POST /api/skillbox/onboarding/import` 消费缓存做选择性导入
- 单元测试用自定义 registry + 临时目录,做到不依赖本机实际目录,稳定可重复

**踩坑(详见 [开发报告.md §2](./开发报告.md#2-问题与方案滚动记录))**

- `NormalizeName` 把已存在的 `-` 当成"待折叠"字符吞掉,已修

**接口**

| Method | Path                                  | 行为                                     |
| ------ | ------------------------------------- | ---------------------------------------- |
| GET    | `/api/skillbox/projects`              | 列表,支持 `?page&size&keyword`           |
| GET    | `/api/skillbox/projects/get`          | 按 id 查(404 if not found)              |
| POST   | `/api/skillbox/projects/create`       | 新建(400 空字段,409 alias/root 冲突)    |
| POST   | `/api/skillbox/projects/update`       | 按 id 更新(部分字段语义)                |
| POST   | `/api/skillbox/projects/delete`       | 按 id 删(404 if not found)              |

**实现要点**

- 业务层 `internal/project.Service` 把 model 包了一层,加 alias / root_path 唯一校验、字段 trim、sentinel error
- controller 每次请求现取 `dbs.GetWriteDb/ReadDb`,避免 init 阶段 db 还没就绪
- 前端用 `@/core/utils/requests` 已有的 http 客户端,无新依赖
- 12 个 service 测试全过(空字段 / 唯一 / 更新冲突 / 分页 / 物理根不强校验)

**踩坑**

- 写 service 时误把 `where.Extra` 写成 `&where.Extra`、又误把 `Page/Size` 当成字段名 → 见 [开发报告.md §2.8](./开发报告.md#28-where-extra-字段名拼错导致编译失败)
- `where.New` 返回的是 `*whereManager`,需 `.Conditions()` 转成 `[]*Condition` 才符合 `append(conds, ...)` 形态
