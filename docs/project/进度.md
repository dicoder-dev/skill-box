**实现要点**

- `internal/skilltester` 内部 3 个独立 check + 1 个聚合 Tester:
  - **static_lint**:frontmatter 必填(name/version/description/triggers)+ 格式正则 + 长度阈值 + 文件路径合法性(`..` / 绝对路径 / ` `)+ SKILL.md 必含 + body 非空 + 硬编码 secret 嗅探(sk- / api_key= / token=)+ trigger 去重
  - **script_runner**:支持自定义 command(拒绝 shell 注入字符)+ skill 自带 `test.sh` / `test.py` / `test.js` / `test`;写到 tempdir 隔离 + 60s timeout + 截断 stdout/stderr 到 32KB
  - **ai_walker**:走 aiengine 流式(默认 `safety_check` preset);无 provider / 无 key / 未知 preset 时降级 skipped,不影响其他 check
- 聚合策略:任一 failed → 整体 failed;任一 errored → 整体 errored(除非有 failed);全部 skipped → 整体 skipped
- `aiengine.Manager.BuildFromConfig(Config)`:service 层只填 Config,manager 内部用 factories 选工厂,避免外部重复 switch kind
- API key 解析走 settings KV(沿用 sai 同款 `secretAdapterForTester`,独立实现一份避免 sai 跨包引用)
- controller 用 `ginp.BindParamsHandler`(同步执行不需要 SSE);`/test/run` 单次执行可能 1-2s(AI 走查 60s 超时),前端按钮显示"测试中…"
- SkillsView 行操作加"测试"按钮(confirm 后触发),编辑器下方加折叠结果面板(status badge + 3 check 行 + JSON 详情折叠)
