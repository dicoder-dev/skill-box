## 文档维护约定

- 新增项目级文档放进本目录,中文文件名 + `.md`,沿用现有 `项目架构.md` / `需求规划.md` / `进度.md` / `开发报告.md` 风格(中文为主,代码 / 接口签名保留英文)
- 业务模块的 API 手册放 [`docs/api/`](../api/),不要混在本目录
- 改动本目录里的文档需要在 PR 描述里写清"对哪些决策有影响",以便回溯
- `需求规划.md` 是设计文档(决策 + 规范),`进度.md` 是状态文档(做了 / 待做),`项目架构.md` 是当前实现说明,`开发报告.md` 是过程文档(为什么这么做 / 跑过什么) — 四者不要重复
- 每次完成一个 12 步里的步骤,**必须**同步更新 `进度.md` 的状态表与"已完成"小节,并在 [开发报告.md](./开发报告.md) 追加"测试报告"小节,再 commit
- 开发中遇到非平凡问题(>5 分钟定位或涉及设计取舍),**必须**追加到 `开发报告.md` 的"问题与方案"小节
- 用户临时加塞、计划外需求 → 写进 `开发报告.md` 的"需求回流"小节,后续并入 `需求规划.md` 修订
