| 文档                                       | 定位                                                                                       | 读者                            |

## 文档维护约定

- 新增项目级文档放进本目录,中文文件名 + `.md`,沿用现有 `项目架构.md` / `需求规划.md` 风格(中文为主,代码 / 接口签名保留英文)
- 业务模块的 API 手册放 [`docs/api/`](../api/),不要混在本目录
- 改动本目录里的文档需要在 PR 描述里写清"对哪些决策有影响",以便回溯
- `需求规划.md` 是活文档:每完成一个 12 步里的步骤,更新对应章节的"状态"列
