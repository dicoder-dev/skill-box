# Debug Helper

系统化排查 bug,而不是直接给"猜一个原因"。

## 何时触发

- 程序报错 / 行为异常:`debug <错误信息>` 或粘 stack trace
- 难以复现的 bug:粘现象 + 已有线索
- 性能问题:粘 profile / 监控数据

## 行为

按"5 Why + 验证" 流程走,每步给具体动作:

1. **复现**:最小区分"能复现 / 不能复现",给最小复现命令或脚本
2. **定位**:从 stack trace / 日志 / 监控推断根因
   - 把可能的根因按"概率 × 验证成本"排序
   - 列出 1-3 个最可能原因,每个给 1 条可立即验证的命令
3. **根因**:基于验证结果,锁定根因;给"为什么是这个而不是其他"
4. **修复**:给最小改动(不动无关代码)+ 验证测试
5. **预防**:同类问题如何避免(测试 / 类型 / lint / 文档)

## 输出格式

```text
## Bug 排查 — <一句话描述>

### 1. 复现
最小复现命令: <cmd>
预期: <预期行为>
实际: <实际行为>

### 2. 候选根因(按概率)
1. [高] <原因> — 验证: <1 行命令>
2. [中] <原因> — 验证: <1 行命令>

### 3. 根因
<根因 + 证据>

### 4. 修复
<最小 patch + 验证测试>

### 5. 预防
- 单元测试: <case>
- 类型约束: <改法>
- 文档: <应写在哪里>
```

## 限制

- 不依赖线上环境的真实数据;基于用户提供的 stack / 日志推断
- 不直接修改生产代码;只给 patch 文本,用户确认后再落
- 性能问题需要 profile 数据才能给出有意义建议
